# 📊 特征名称解释指南

当你查看 `feature_importance_random_forest.png` 时，看到的特征名称遵循以下命名规则：

## 🔤 特征名称格式

```
{传感器类型}_{轴}_{特征名}
```

例如：
- `ACC_x_mean` = 加速度计的X轴的平均值
- `GYRO_z_std` = 陀螺仪的Z轴的标准差
- `ROT_VEC_w_fft_energy` = 旋转矢量的W分量的FFT能量

---

## 📱 传感器类型（第一部分）

| 代码 | 含义 | 测量什么 |
|------|------|----------|
| `ACC` | Accelerometer（加速度计） | 手机在空间中的加速度，包括重力 |
| `GYRO` | Gyroscope（陀螺仪） | 手机的旋转速度（角速度） |
| `ROT_VEC` | Rotation Vector（旋转矢量） | 手机的3D方向（四元数表示） |
| `MAG` | Magnetometer（磁力计） | 周围的磁场强度 |

**为什么这些传感器能预测密码？**
- 当你按不同数字键时，手机的倾斜角度、加速度、旋转都会略有不同
- 按键位置不同（0在底部中间，9在右上角）导致传感器读数不同

---

## 🧭 轴/分量（第二部分）

| 轴 | 含义 | 适用传感器 |
|----|------|-----------|
| `x` | X轴（水平左右） | ACC, GYRO, ROT_VEC, MAG |
| `y` | Y轴（水平前后） | ACC, GYRO, ROT_VEC, MAG |
| `z` | Z轴（垂直上下） | ACC, GYRO, ROT_VEC, MAG |
| `w` | W分量（四元数标量部分） | 仅 ROT_VEC |

**坐标系说明**：
```
      Y (手机顶部方向)
      ↑
      |
      |
      └────→ X (手机右侧方向)
     /
    /
   Z (手机屏幕朝外方向)
```

---

## 📈 时域特征（第三部分 - 统计特征）

这些特征描述传感器数据在按键期间的统计特性：

| 特征名 | 含义 | 公式/解释 |
|--------|------|----------|
| `mean` | 平均值 | 数据的中心趋势 |
| `std` | 标准差 | 数据的波动程度 |
| `min` | 最小值 | 数据的最低点 |
| `max` | 最大值 | 数据的最高点 |
| `median` | 中位数 | 排序后的中间值 |
| `range` | 范围 | max - min，数据跨度 |
| `rms` | 均方根 | √(平均值²)，能量度量 |
| `skewness` | 偏度 | 数据分布的对称性（负=左偏，正=右偏） |
| `kurtosis` | 峰度 | 数据分布的尖锐程度 |
| `q25` | 第一四分位数 | 25%的数据小于此值 |
| `q75` | 第三四分位数 | 75%的数据小于此值 |
| `zero_crossing_rate` | 过零率 | 数据穿过零点的频率 |

**示例解释**：
- `ACC_x_mean = 2.5` → 按键期间，X轴平均加速度为2.5 m/s²
- `GYRO_z_std = 0.8` → Z轴旋转速度波动较大（标准差0.8）
- `ACC_y_range = 5.2` → Y轴加速度变化了5.2 m/s²（从最小到最大）

---

## 🌊 频域特征（第三部分 - FFT特征）

这些特征描述传感器数据的频率成分（经过FFT变换）：

| 特征名 | 含义 | 解释 |
|--------|------|------|
| `fft_energy` | FFT能量 | 频谱的总能量 |
| `fft_mean` | FFT平均值 | 频谱幅值的平均值 |
| `fft_std` | FFT标准差 | 频谱幅值的波动 |
| `fft_max` | FFT最大值 | 最强频率分量的幅值 |
| `dominant_frequency` | 主频率 | 能量最强的频率（Hz） |
| `dominant_magnitude` | 主频幅值 | 主频率的能量大小 |
| `spectral_centroid` | 频谱质心 | 频谱的"重心"位置 |

**为什么需要频域特征？**
- 不同的按键动作有不同的"节奏"和"震动模式"
- 频域特征能捕捉到这些微妙的震动差异

**示例解释**：
- `ACC_z_fft_energy = 125.3` → Z轴震动总能量为125.3
- `GYRO_x_dominant_frequency = 3.5` → X轴旋转的主要频率是3.5Hz
- `ROT_VEC_y_spectral_centroid = 7.2` → 频谱能量集中在7.2Hz附近

## 🎯 特征重要性图怎么看？

当你打开 `feature_importance_random_forest.png` 时：

1. **Y轴（左侧）**：特征名称
2. **X轴（底部）**：重要性分数（0-1之间）
3. **条形长度**：越长 = 该特征对预测越重要

### 示例分析

假设图中显示：
```
ACC_z_mean          ████████████████████ 0.15
GYRO_x_std          ████████████████     0.12
ROT_VEC_y_fft_energy███████████          0.08
ACC_x_mean          ██████                0.05
...
```

**解读**：
- **最重要**：`ACC_z_mean`（加速度计Z轴平均值）权重0.15
  - 说明：按不同数字时，**垂直方向的加速度变化**最能区分数字
  - 原因：可能是手的高度变化（按9比按0手抬得更高）

- **第二重要**：`GYRO_x_std`（陀螺仪X轴标准差）权重0.12
  - 说明：**水平旋转的波动程度**很有用
  - 原因：按左边数字(1,4,7)和右边数字(3,6,9)时旋转不同

- **第三重要**：`ROT_VEC_y_fft_energy`（旋转矢量Y轴FFT能量）权重0.08
  - 说明：**前后方向的震动能量**有一定作用
  - 原因：按上方(7,8,9)和下方(0,1,2,3)数字时震动模式不同

## 💡 实际意义

### 高重要性特征说明什么？

如果你的模型显示以下特征最重要：

1. **ACC相关特征高** → 说明**手的加速度变化**最能区分不同数字
   - 物理意义：按键位置导致手臂移动方式不同

2. **GYRO相关特征高** → 说明**手机旋转/倾斜**最有用
   - 物理意义：按不同位置的键时，拇指施力导致手机微旋转

3. **ROT_VEC相关特征高** → 说明**手机整体方向变化**很关键
   - 物理意义：综合了加速度和陀螺仪信息，捕捉整体姿态

4. **频域特征(fft_*)高** → 说明**震动的频率特性**很重要
   - 物理意义：按键的"节奏"和"力度"有特定模式

### 低重要性特征说明什么？

- 该传感器/轴/特征对区分数字贡献不大
- 可以考虑去除这些特征来简化模型

---

## 🔍 如何利用这些信息？

### 1. 改进数据收集
如果 `ACC_z_mean` 很重要：
- 保持手机持握方式一致
- 避免在按键时晃动手机

### 2. 特征工程
- 可以添加更多 **高重要性传感器** 的特征
- 删除重要性很低（<0.01）的特征

### 3. 理解隐私风险
- 如果仅用3-4个特征就能达到70%准确率 → **隐私风险很高**
- 说明攻击者不需要太多信息就能推测密码

## 📝 完整示例

假设特征重要性图显示：

```
特征名                    重要性   解释
------------------------------------------------------------------
ACC_z_std               0.156    Z轴加速度波动最关键
GYRO_x_mean             0.142    X轴旋转平均值第二重要
ROT_VEC_y_fft_energy    0.089    Y轴频域能量有用
ACC_x_range             0.067    X轴加速度变化范围
MAG_z_median            0.053    磁场Z轴中位数
...（更多特征）
```

**你可以得出**：
1. ✅ 加速度计是最有用的传感器（ACC_z_std最高）
2. ✅ 垂直方向(Z轴)和水平左右(X轴)的运动最能区分数字
3. ✅ 陀螺仪也很重要，说明手机旋转是关键线索
4. ✅ 频域特征有贡献，说明按键的动态过程（不仅是静态值）很重要
5. ⚠️ 磁力计重要性较低，可能不太必要
